Домашнее задание 26:
1. Решите задачу по проектированию системы IoT: Вы разрабатываете модель данных для системы "умного дома", которая собирает данные о температуре, влажности и потреблении энергии. Какую модель данных вы бы использовали, и почему?
(Ответ предоставьте в 3-4 предложения)
2. Задача по выбору модели данных для банковского приложения: Сравните графовую и реляционную модели данных для системы анализа транзакций. Опишите, в каких случаях каждая из них будет наиболее эффективной и почему.
(Ответ предоставьте в 3-4 предложения)


1. Для хранения данных о температуре, влажности и потреблении энергии можно выбраь временную (time-series) модель данных, т.к. данные от датчиков поступают регулярно по времени. Это позволит эффективно хранить и анализировать данные с првиязкой к временным меткам. Для хранения метаданных утсройств, их конфигураций можно использовать документоориентированную модель данных, обеспечивая гибкость и быстрый доступ к данным.

2. Реляционная модель данных будет эффективной для хранения информации и пользователях,счетах, для стандартных транзакций, обеспечивая целостность и согласованность данных через ACID-транзакции. Графовая модель больше походит для анализа взаимосвязей между транзакциями (для выявления мошеннических схем), клиентами и их счетами. Она позволяет быстро выявлять подозрительные схемы, отслеживая цепочки переводов и связи между пользователями. Можно использовать гибридный поход, где реляционная бд хранит ключевые финансовые данные, а графовая анализиоует связи.
